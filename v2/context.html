1.1. Цель системы

Целью системы является снижение нагрузки на регистраторов и уменьшение времени ожидания заявителей за счёт предварительной записи на услуги, автоматизированного управления временными слотами, поддержки электронной очереди и уведомления пользователей о создании, изменении или отмене записи.

Система должна обеспечивать корректное распределение заявителей по временным интервалам, предотвращать двойную запись на один и тот же слот, учитывать расписание подразделений и окон обслуживания, а также предоставлять сотрудникам актуальную информацию о текущей очереди и загрузке.

1.4. Границы системы

Внутри SoI

В состав системы входят:

  • публичный веб-интерфейс заявителя;
  • интерфейс сотрудника и администратора расписаний;
  • API Gateway / Backend API;
  • сервис записи на приём;
  • сервис расписаний и расчёта слотов;
  • сервис электронной очереди;
  • модуль уведомлений;
  • модуль отчётности и аудита;
  • основная база данных;
  • кэш и механизм краткоживущих блокировок;
  • очередь сообщений для событий записи и уведомлений;
  • API для терминалов и табло.

Вне SoI

За пределами системы находятся:

  • корпоративный SSO / IdP;
  • внешний SMS/e-mail/push-шлюз;
  • внешняя календарная система;
  • физическое оборудование терминалов и табло;
  • внешние средства мониторинга;
  • BI-система организации;
  • организационные регламенты МФЦ;
  • фактическая работа сотрудников с заявителями вне интерфейса системы.

1.6. Допущения и ограничения

  1. Система рассматривается в контексте одного МФЦ с возможностью расширения на несколько подразделений.
  2. Заявитель может использовать веб-интерфейс без корпоративной учётной записи сотрудника.
  3. Сотрудники входят в систему через SSO.
  4. Уведомления отправляются через внешний сервис, который может быть временно недоступен.
  5. Расписание может задаваться вручную администратором или синхронизироваться с внешним календарём.
  6. Система должна сохранять корректность записи даже при одновременных запросах нескольких пользователей на один слот.
  7. Терминалы и табло считаются внешним оборудованием; в SoI входят только API и логика взаимодействия с ними.
  8. Персональные данные заявителей должны обрабатываться с учётом требований защиты информации и минимизации доступа.
